    The team conducted meetings to ensure that the individual web app maintains the same look and feel. Responsiveness and mobile phone optimization were also considered; three different layouts were set. The first layout is for desktop or laptop displays. In this layout, all the widgets and map are in view, and none can be hidden. The sidebar widget was used for the tablet layout. The text, chart, and legend were all inside the side panel, which can be hidden from view when interacting with the map. The final layout is for small screen devices or mobile phones. The team decided that displaying the dynamic text is sufficient for this view. All the other widgets were moved to the pending list.

Encountered Limitations

Experience Builder Formatting

When experimenting with the mobile UI setup, it was noticed that certain formatting attributes were shared amongst the various screen setups (desktop, tablet, mobile) and this made it more difficult when you could not adjust these attributes uniquely to each of the different setups (if required). Sometimes it would be the case where you would want the formatting attributes to be shared, but other times they should be able to be 'unlocked' so that they can be set to their own unique value based on the screen setup (resolution or screen size).

Janelli Powell

Collaborating in Experience Builder

ArcGIS Experience Builder being an online-based web app solution, does not have any collaborative functionality. Users in the same ArcGIS Online organization cannot edit a user’s web experience. The only workaround I found was to copy the item ID of the web app to be edited, open it using this URL https://experience.arcgis.com/builder/?id={INSERT ItemID Here}, save to my contents, and start editing. Passing the edited content back has the same workflow. Specifications or meetings are guaranteed to be needed when collaborating in Experience Builder.

Vince Ruel Alonte

Experience Builder Editing and Data Connection

ArcGIS Experience Builder provides lots of modification and data connection options for the developers. However since it is a web app, it still has limitations in modification and data connection. For example, when the developer wants to put a legend for the data, the process of connecting data is complicated and not intuitive, also the developer can not modify the legend components.

Jeonghyeok Seo
